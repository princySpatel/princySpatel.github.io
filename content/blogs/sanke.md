## Overview
This project is a standalone implementation of the classic Snake game. I built this early on by following architectural tutorials to get a practical, hands-on introduction to **Java** and **Object-Oriented Programming (OOP)**. 

While I am still expanding my Java expertise, this project served as a fantastic exercise in structuring code properly and managing a project from a blank script all the way to a compiled `.exe` executable.

## Core Concepts Explored
Building this game helped solidify several foundational software engineering concepts rather than just writing a simple script:

* **The Core Game Loop:** Implemented a continuous cycle to handle snake movement, listen for keyboard input, and randomly generate food coordinates.
* **Object-Oriented Design:** Structured the game using distinct classes and objects to separate the logic of the Snake, the Food, and the Game Board UI.
* **State Management:** Developed conditional logic to transition smoothly between active gameplay and a definitive "Game Over" screen upon collision with the walls or the snake's own tail.
* **Visual Grid Rendering:** Utilized Java's built-in graphics libraries to draw a clean, responsive grid system and track the player's score dynamically on the screen.
